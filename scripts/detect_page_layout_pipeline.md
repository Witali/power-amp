# OpenCV page-layout pipeline

Этот документ описывает текущий пайплайн разметки страниц журналов рядом с основным скриптом `detect_page_layout.py`.

## Основной запуск

Типовой запуск одной страницы:

```powershell
python scripts\detect_page_layout.py --image study\opencv_layout_regression_pages\sources\b.1997-04.018.jpg --out-dir .tmp\page_layout --preview-width 1100 --frequency-hints validate
```

Пакетный отчет по регрессионным страницам:

```powershell
python scripts\generate_opencv_layout_report.py
```

## Последовательность обработки

1. `detect_page_layout.py` читает исходную страницу через OpenCV и уменьшает ее до рабочего размера `max_analysis_side`.
2. Страница переводится в grayscale, строится foreground mask через Otsu threshold, затем отдельно строятся Canny edges.
3. `make_block_mask()` объединяет текстоподобные морфологические закрытия и edge-блоки, чтобы получить первичные кандидаты областей.
4. `detect_candidate_boxes()` очищает поля страницы, находит контуры, объединяет близкие прямоугольники, режет большие области по вертикальным и горизонтальным белым коридорам, а также отделяет цветные боковые полосы.
5. Для крупных смешанных областей дополнительно ищутся вложенные line-art блоки: схемы, диаграммы, таблицы и PCB. Крупные области с сигнатурами радиокомпонентов и осевыми линиями сохраняются как `schematic/circuit`, даже если подписи внутри дают высокий текстовый score; боковые панели с подписями и рамкой могут присоединяться к соседней крупной схеме. После этого текст восстанавливается вокруг найденных визуальных блоков.
6. Если включен `--frequency-hints validate`, `layout_frequency.py` строит частотные и гистограммные подсказки. В режиме `validate` они используются как проверка и предупреждения; в режиме `hints` могут добавлять кандидаты визуальных блоков.
7. Для каждого кандидата `feature_dict()` считает признаки: размер, плотность чернил, edge density, насыщенность, оценки строк текста, горизонтальные/вертикальные линии, сигнатуры компонентов и PCB.
8. `classify_features()` объединяет маленькую OpenCV ANN_MLP модель с эвристическими `rule_scores()`. Классы сейчас: `text`, `heading`, `image`, `schematic/circuit`, `diagram`, `pcb`, `table`, `other`. Для годовых оглавлений есть отдельные проверки плотных текстовых списков и широких заголовков с линейками, чтобы они не уходили в `image` или `schematic/circuit`; чистые длинные горизонтальные разделители без вертикальных линий и компонентной сигнатуры не повышаются до `heading`; цифровые временные диаграммы с одной доминирующей горизонтальной осью не повышаются до `text` или `schematic/circuit`; жирные display-заголовки и монохромные иконки дополнительно отделяются от схемных/PCB-like признаков.
9. После классификации выполняется постобработка: разбиение текстовых колонок, выделение внутренних display-заголовков из крупных текстовых блоков, объединение фрагментированных строк годовых оглавлений, отделение заголовка оглавления по горизонтальному пустому коридору с сохранением этого коридора как зазора между рамками, слияние соседних фрагментов одного многострочного заголовка, повторное разбиение таких блоков на колонки, слияние узких колонок номеров выпусков/страниц с соседним текстом, слияние соседних фрагментов одной колонки оглавления, повторное слияние оставшихся колонок номеров, объединение связанных схем без присоединения waveform-диаграмм как line-art attachments, объединение стеков диаграмм, объединение фрагментов картинок без захвата waveform-диаграмм, возврат плотных списков оглавления из `image` обратно в `text`, подавление слабых вложенных визуальных дублей, чистых горизонтальных разделителей, слабых пустых `heading`-полос, текстовых фрагментов меньше настроенного минимума в буквах/пикселях с сохранением плотных многострочных блоков, мелких служебных блоков, нижних колонтитулов оглавления и цветных визуальных артефактов у боковых полей страницы.
10. `resolve_block_overlaps()` устраняет пересекающиеся области: для зоны пересечения заново считаются признаки, выбирается владелец, а проигравший блок получает вырез в outline.
    Для стековых waveform-диаграмм ранний текстовый вырез отключается, а спор с соседним текстом решается на этом шаге: если пересечение содержит плотные осевые линии, область остается у диаграммы.
11. `attach_caption_candidates()` ищет подписи `Рис.`, `Figure` и похожие подписи рядом с визуальными блоками; на preview они подсвечиваются желтым.
12. Скрипт пишет `layout.json`, опциональные crops по типам блоков и `preview.png` с цветной разметкой.

## Связанные файлы

- `scripts/layout_config.py` - классы, цвета и основные пороги.
- `scripts/layout_component_signatures.py` - признаки радиокомпонентов, PCB-дорожек и площадок.
- `scripts/layout_frequency.py` - частотный, гистограммный и кластерный hint layer.
- `scripts/generate_opencv_layout_report.py` - HTML-отчет по набору регрессионных страниц.
- `study/opencv_layout_regression_pages/manifest.json` - список страниц для регрессионного отчета.
- `config/pipeline_parallelism.json` - общий лимит параллельных OpenCV/OCR задач; отчет использует
  `max_parallel_opencv_tasks`, если не передан `--max-parallel-opencv`.
- `tests/test_detect_page_layout.py` - модульные тесты классификации, геометрии, overlaps и постобработки.
